# Componenta HTTP Emitter

Отправщик PSR-7 ответов для HTTP-приложений Componenta. Он отправляет статус, заголовки и тело ответа в PHP output.

Используйте пакет во внешней HTTP-точке входа после того, как PSR-15 конвейер вернул `ResponseInterface`.

## Граница пакета

Пакет только отправляет уже построенный ответ. Он не строит ответы, не создает серверные запросы и не запускает промежуточные обработчики. Для построения ответов используйте `componenta/http-responder`, а для HTTP-конвейера приложения — `componenta/app-http`.

## Установка

```bash
composer require componenta/http-emitter
```

`Componenta\Http\EmitterConfigProvider` публикуется через метаданные Composer.

## Основной API

`EmitterInterface::emit(ResponseInterface $response): void` отправляет ответ. `Emitter` — стандартная реализация.

```php
use Componenta\Http\EmitterInterface;

/** @var EmitterInterface $emitter */
$emitter->emit($response);
```

`Emitter` принимает необязательный размер чанка:

```php
$emitter = new Componenta\Http\Emitter(chunkSize: 16_384);
```

`EmitStrategy` управляет отправкой тела:

| Вариант | Значение |
|---|---|
| `Full` | Отправить все тело ответа. |
| `Partial` | Отправить тело ответа с диапазоном байтов. |
| `NoBody` | Отправить заголовки без тела. |

`EmitterException` выбрасывается при ошибках отправщика.

## Поведение во время выполнения

Перед отправкой `Emitter` проверяет, что заголовки еще не отправлены, и очищает активные буферы вывода. Затем он выбирает стратегию по статусу и заголовкам ответа:

| Стратегия | Когда используется |
|---|---|
| `NoBody` | Информационные статусы, `204`, `304`, `416` или неудовлетворимый `Content-Range`. |
| `Partial` | Статус `206` с заголовком `Content-Range`. |
| `Full` | Все остальные ответы. |

Для полных ответов отправщик добавляет `Content-Length`, если размер потока известен и заголовок отсутствует. Для seekable-потоков с известным размером он также добавляет `Accept-Ranges: bytes`. Тело отправляется чанками и прекращается раньше, если клиент отключился.

Для частичных ответов корректный `Content-Range` управляет диапазоном байтов, который будет отправлен, если поток тела поддерживает seek. Если диапазон нельзя разобрать или поток не поддерживает seek, отправщик отправляет все тело, которое уже находится в ответе.

## Конфигурация

Провайдер регистрирует `Emitter` и псевдоним `EmitterInterface` на него. Обычно приложение получает отправщик из контейнера.
